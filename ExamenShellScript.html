    Examen ShellScript - ASO¶

    Curso: Administración de Sistemas Operativos
    Módulo: ShellScript
    Duración: 2 horas
    Puntuación total: 60 puntos

    Instrucciones Generales¶

    • Tiempo: 2 horas (120 minutos)
    • Puntuación: 60 puntos total
    • Formato: Resolver los ejercicios creando scripts en ShellScript
    • Entrega: Crear un directorio examen_[nombre]_[apellidos] con todos los scripts
    • Nomenclatura: Los scripts deben llamarse ejercicio1.sh, ejercicio2.sh, etc.
    • Documentación: Cada script debe incluir comentarios explicativos
    • Pruebas: Verificar que todos los scripts funcionan correctamente

    Recursos Permitidos¶

    Recursos Permitidos

    Durante el examen SOLO se permite utilizar:

    • Apuntes del tema ShellScript
    • Aules (plataforma educativa)
    • Terminal de bash de LliureX (para probar los scripts)

    NO se permite:

    • Internet (excepto para acceder a Aules)
    • Documentación externa
    • Ayuda de compañeros
    • Dispositivos móviles
    • Cualquier otro recurso no autorizado

    Ejercicio 1: Script Básico (8 puntos)¶

    Crea un script llamado ejercicio1.sh que:

    1. Muestre un mensaje de bienvenida con el nombre del usuario actual
    2. Muestre la fecha y hora actual del sistema
    3. Muestre información del sistema (nombre del host, sistema operativo)
    4. Cuente el número de archivos en el directorio actual
    5. Muestre el espacio libre en disco

    Criterios de evaluación:

    • Uso correcto de variables (2 puntos)
    • Comandos del sistema (2 puntos)
    • Salida formateada (2 puntos)
    • Comentarios y estructura (2 puntos)

    Ejercicio 2: Gestión de Parámetros (10 puntos)¶

    Crea un script llamado ejercicio2.sh que:

    1. Reciba exactamente 3 parámetros por línea de comandos
    2. Valide que se hayan pasado los 3 parámetros (si no, mostrar error y salir)
    3. Muestre información sobre los parámetros:
    • Número total de parámetros
    • Nombre del script
    • Los 3 parámetros introducidos
    • Los parámetros en orden inverso
    1. Realice operaciones matemáticas con los parámetros (asumiendo que son números):
    • Suma de los 3 números
    • Producto de los 3 números
    • Media aritmética

    Ejemplo de uso: 

    ./ejercicio2.sh 5 10 15

    Criterios de evaluación:

    • Validación de parámetros (3 puntos)
    • Uso de variables especiales (2 puntos)
    • Operaciones aritméticas (3 puntos)
    • Manejo de errores (2 puntos)

    Ejercicio 3: Estructuras Condicionales (12 puntos)¶

    Crea un script llamado ejercicio3.sh que implemente un sistema de calificaciones:

    1. Pida al usuario que introduzca una nota (número entero del 0 al 10)
    2. Valide la entrada (debe ser un número entre 0 y 10)
    3. Use una estructura case para mostrar la calificación según la nota:
    • 0-4: "SUSPENSO"
    • 5: "SUFICIENTE"
    • 6: "BIEN"
    • 7-8: "NOTABLE"
    • 9-10: "SOBRESALIENTE"
    1. Añada funcionalidad extra:
    • Si la nota es 10, mostrar "¡FELICITACIONES!"
    • Si la nota es menor que 5, mostrar "Necesitas estudiar más"
    • Mostrar el porcentaje correspondiente a la nota

    Criterios de evaluación:

    • Estructura case correcta (4 puntos)
    • Validación de entrada (3 puntos)
    • Funcionalidades adicionales (3 puntos)
    • Interfaz de usuario (2 puntos)

    Ejercicio 4: Bucles y Arrays (15 puntos)¶

    Crea un script llamado ejercicio4.sh que:

    1. Cree un array con los nombres de 5 ciudades
    2. Use un bucle for para mostrar todas las ciudades
    3. Implemente un menú con las siguientes opciones:
    • Mostrar ciudades en orden normal
    • Mostrar ciudades en orden inverso
    • Mostrar ciudades ordenadas alfabéticamente
    • Buscar una ciudad específica
    • Mostrar estadísticas (número total, primera, última)
    • Salir
    1. Use select para el menú interactivo
    2. Implemente todas las opciones usando funciones

    Criterios de evaluación:

    • Creación y uso de arrays (4 puntos)
    • Implementación del menú con select (4 puntos)
    • Funciones para cada opción (4 puntos)
    • Búsqueda y ordenación (3 puntos)

    Ejercicio 5: Gestión de Archivos (10 puntos)¶

    Crea un script llamado ejercicio5.sh que:

    1. Cree un archivo de texto llamado datos.txt con información de usuarios
    2. Formato del archivo: nombre#edad#ciudad#telefono
    3. Implemente las siguientes funciones:
    • añadir_usuario(): Añadir nuevo usuario
    • listar_usuarios(): Mostrar todos los usuarios
    • buscar_usuario(): Buscar por nombre
    • usuarios_por_ciudad(): Mostrar usuarios de una ciudad
    • estadisticas(): Mostrar estadísticas (total usuarios, edad media)
    1. Use un menú para acceder a las funciones
    2. Valide las entradas del usuario

    Ejemplo de datos: 

    Juan#25#Madrid#123456789
María#30#Barcelona#987654321
Pedro#22#Valencia#555666777

    Criterios de evaluación:

    • Gestión de archivos (3 puntos)
    • Funciones implementadas (4 puntos)
    • Validación de datos (2 puntos)
    • Interfaz de usuario (1 punto)

    Ejercicio 6: Script de Monitoreo (5 puntos)¶

    Crea un script llamado ejercicio6.sh que:

    1. Monitoree el estado de 3 servicios simulados (usando las IPs: 8.8.8.8, 1.1.1.1, 127.0.0.1)
    2. Use ping para verificar la conectividad
    3. Genere un informe con:
    • Estado de cada servicio (activo/inactivo)
    • Tiempo de respuesta
    • Porcentaje de disponibilidad
    1. Guarde el informe en un archivo informe_$(date +%Y%m%d_%H%M%S).log
    2. Muestre un resumen por pantalla

    Criterios de evaluación:

    • Comando ping y redirecciones (2 puntos)
    • Generación de informe (2 puntos)
    • Formato de salida (1 punto)

    Criterios de Evaluación General¶

    Funcionalidad (40 puntos)¶

    • Ejercicio 1: 8 puntos
    • Ejercicio 2: 10 puntos
    • Ejercicio 3: 12 puntos
    • Ejercicio 4: 15 puntos
    • Ejercicio 5: 10 puntos
    • Ejercicio 6: 5 puntos

    Calidad del Código (15 puntos)¶

    • Comentarios: 5 puntos
    • Estructura y organización: 5 puntos
    • Buenas prácticas: 5 puntos

    Documentación (5 puntos)¶

    • README.md con instrucciones de uso
    • Explicación de cada script
    • Ejemplos de ejecución

    Instrucciones de Entrega¶

    1. Crear directorio: examen_[nombre]_[apellidos]
    2. Incluir todos los scripts con nombres correctos
    3. Añadir archivo README.md con: - Descripción de cada ejercicio - Instrucciones de ejecución - Capturas de pantalla (opcional)
    4. Comprimir en ZIP el directorio completo
    5. Subir al Moodle antes del tiempo límite

    Ejemplo de Estructura de Entrega¶

    examen_juan_garcia/
├── ejercicio1.sh
├── ejercicio2.sh
├── ejercicio3.sh
├── ejercicio4.sh
├── ejercicio5.sh
├── ejercicio6.sh
├── README.md
└── datos.txt (generado por ejercicio5.sh)

    Notas Importantes¶

    • Recursos: Recuerda que solo puedes usar los apuntes, Aules y el terminal de LliureX
    • Tiempo: Gestiona bien el tiempo, cada ejercicio tiene diferente dificultad
    • Pruebas: Verifica que todos los scripts funcionan antes de entregar
    • Comentarios: Incluye comentarios explicativos en cada script
    • Validación: Implementa validaciones de entrada donde sea necesario
    • Errores: Maneja posibles errores de forma adecuada
    • Integridad: El uso de recursos no autorizados supondrá la anulación del examen
