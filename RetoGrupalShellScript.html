    RETO GRUPAL - Sistema de Gestión de Servidores (30 puntos)¶

    RETO GRUPAL: Sistema de Gestión de Servidores

    Valoración: 30 puntos
    Modalidad: Trabajo en grupos de 3-4 personas
    Tiempo estimado: 4 sesiones (8 horas)
    Fecha límite: Sesión 8 (Presentación y evaluación)

    Descripción del Reto¶

    Debéis crear un Sistema de Gestión de Servidores completo en ShellScript que permita a un administrador de sistemas gestionar múltiples servidores de forma automatizada. El sistema debe integrar todos los conceptos aprendidos durante la unidad.

    Requisitos del Sistema¶

    1. Estructura del Proyecto (5 puntos)¶

    • Crear un directorio principal sistema_gestion_servidores/

    • Organizar el código en módulos separados:

    • menu_principal.sh - Menú principal del sistema

    • funciones_servidor.sh - Funciones para gestión de servidores
    • monitoreo.sh - Scripts de monitoreo
    • backup.sh - Sistema de copias de seguridad
    • logs.sh - Gestión de logs
    • configuracion.conf - Archivo de configuración

    2. Menú Principal Interactivo (5 puntos)¶

    • Implementar un menú con select que incluya:

    • Gestión de servidores

    • Monitoreo del sistema
    • Copias de seguridad
    • Gestión de logs
    • Configuración
    • Salir
    • Cada opción debe llamar a funciones específicas
    • Validación de entrada del usuario

    3. Gestión de Servidores (8 puntos)¶

    • Archivo de configuración (servidores.txt) con formato: 
      nombre_servidor#ip#puerto#estado#descripcion
    • Funciones requeridas:
    • añadir_servidor() - Añadir nuevo servidor
    • listar_servidores() - Mostrar todos los servidores
    • buscar_servidor() - Buscar por nombre, IP o estado
    • modificar_servidor() - Editar información de servidor
    • eliminar_servidor() - Borrar servidor
    • ordenar_servidores() - Ordenar alfabéticamente

    4. Sistema de Monitoreo (6 puntos)¶

    • Función monitorear_servidores():
    • Leer lista de servidores desde archivo
    • Usar ping para verificar conectividad
    • Generar informe de estado (activo/inactivo)
    • Guardar resultados en estado_servidores.log
    • Función estadisticas_sistema():
    • Mostrar información del sistema local
    • Contar servidores activos/inactivos
    • Calcular porcentaje de disponibilidad

    5. Sistema de Copias de Seguridad (4 puntos)¶

    • Función crear_backup():
    • Crear directorio de backup con fecha
    • Copiar archivos de configuración
    • Comprimir en archivo .tar.gz
    • Generar log de backup
    • Función restaurar_backup():
    • Listar backups disponibles
    • Permitir selección y restauración

    6. Gestión de Logs (2 puntos)¶

    • Función ver_logs():
    • Mostrar logs del sistema
    • Filtrar por fecha o tipo
    • Contar líneas de error
    • Función limpiar_logs():
    • Archivar logs antiguos
    • Limpiar logs vacíos

    Criterios de Evaluación¶

    Criterio Puntos Descripción
    Funcionalidad 15 El sistema funciona correctamente y cumple todos los requisitos
    Estructura del código 5 Código bien organizado, comentado y modular
    Uso de conceptos 5 Aplicación correcta de variables, bucles, funciones, vectores
    Documentación 3 README.md con instrucciones de uso
    Creatividad 2 Funcionalidades adicionales o mejoras innovadoras

    Entregables¶

    1. Código fuente completo en directorio sistema_gestion_servidores/
    2. README.md con:
    • Descripción del proyecto
    • Instrucciones de instalación y uso
    • Explicación de cada módulo
    • Capturas de pantalla del funcionamiento 3. Demo en clase (5 minutos por grupo)

    Funcionalidades Adicionales (Puntos Extra)¶

    • +2 puntos: Implementar autenticación de usuarios
    • +2 puntos: Crear gráficos ASCII con estadísticas
    • +2 puntos: Notificaciones por email cuando un servidor falla
    • +1 punto: Interfaz con colores usando tput
    • +1 punto: Sistema de alertas sonoras

    Simulación de Servidores¶

    Para probar el sistema sin necesidad de servidores reales, podéis usar las siguientes técnicas:

    Opción 1: Servidores Simulados con nc (netcat)¶

    # Simular servidor web en puerto 8080
# nc = netcat, herramienta para leer y escribir datos a través de la red
# -l = modo "listen" (escuchar), convierte nc en un servidor
# 8080 = puerto donde escuchará el servidor (puerto común para servidores web)
# & = ejecuta el comando en segundo plano (background)
nc -l 8080 &

# Simular servidor SSH en puerto 2222
# SSH normalmente usa puerto 22, aquí usamos 2222 para evitar conflictos
# Este comando crea un "servidor" que escucha en el puerto 2222
nc -l 2222 &

# Simular servidor de base de datos en puerto 3306
# Puerto 3306 es el puerto estándar para MySQL
# nc escuchará conexiones entrantes en este puerto
nc -l 3306 &

    Opción 2: Servidores Simulados con ShellScript¶

    # Crear script simulador_shell.sh
#!/bin/bash
# Shebang: indica que el script debe ejecutarse con bash

# Simular servidor HTTP simple con netcat
simular_servidor_web() {
    # function = define una función llamada simular_servidor_web
    # { = inicio del bloque de código de la función

    echo "Iniciando servidor web simulado en puerto 8080..."
    # echo = muestra un mensaje informativo por pantalla

    while true; do
        # while true = bucle infinito que se ejecuta continuamente
        # do = inicio del bloque de código del bucle

        echo -e "HTTP/1.1 200 OK\r\n\r\nServidor Web Simulado - $(date)" | nc -l 8080
        # echo -e = activa la interpretación de caracteres de escape (\r\n)
        # "HTTP/1.1 200 OK" = respuesta HTTP válida (protocolo web)
        # \r\n = retorno de carro y nueva línea (formato HTTP)
        # \r\n\r\n = separa cabeceras del cuerpo de la respuesta
        # "Servidor Web Simulado - $(date)" = mensaje personalizado con fecha actual
        # $(date) = sustitución de comando que ejecuta date y usa su salida
        # | = tubería que envía la salida de echo a nc
        # nc -l 8080 = netcat en modo servidor escuchando en puerto 8080

    done &
    # done = fin del bloque de código del bucle
    # & = ejecuta el bucle completo en segundo plano

}
# } = fin del bloque de código de la función

# Simular servidor SSH
simular_servidor_ssh() {
    # function = define una función llamada simular_servidor_ssh

    echo "Iniciando servidor SSH simulado en puerto 2222..."
    # Mensaje informativo para el usuario

    while true; do
        # Bucle infinito para mantener el servidor activo

        echo "SSH-2.0-OpenSSH_7.4 Simulado" | nc -l 2222
        # "SSH-2.0-OpenSSH_7.4" = banner típico de un servidor SSH real
        # "Simulado" = indica que es una simulación
        # nc -l 2222 = escucha en puerto 2222 (evita conflicto con puerto 22)

    done &
    # Ejecuta el bucle en segundo plano

}

# Simular servidor de base de datos
simular_servidor_db() {
    # function = define una función llamada simular_servidor_db

    echo "Iniciando servidor MySQL simulado en puerto 3306..."
    # Mensaje informativo

    while true; do
        # Bucle infinito para mantener el servidor activo

        echo "MySQL Simulado - Puerto 3306" | nc -l 3306
        # "MySQL Simulado" = mensaje que simula respuesta de MySQL
        # "Puerto 3306" = indica el puerto estándar de MySQL
        # nc -l 3306 = escucha en puerto 3306

    done &
    # Ejecuta el bucle en segundo plano

}

        # | = tubería que envía la salida de echo a nc
        # nc -l 8080 = netcat en modo servidor escuchando en puerto 8080

    Opción 5: Simulación Avanzada con ShellScript¶

    # Crear script simulador_avanzado.sh
#!/bin/bash
# Shebang: indica que el script debe ejecutarse con bash

# Función para simular servidor con respuesta personalizada
simular_servidor_personalizado() {
    # function = define una función llamada simular_servidor_personalizado
    # { = inicio del bloque de código de la función

    local puerto=$1
    # local = declara una variable local dentro de la función
    # puerto = nombre de la variable que contendrá el puerto
    # $1 = primer parámetro pasado a la función

    local nombre=$2
    # local = declara una variable local dentro de la función
    # nombre = nombre de la variable que contendrá el nombre del servidor
    # $2 = segundo parámetro pasado a la función

    local respuesta=$3
    # local = declara una variable local dentro de la función
    # respuesta = nombre de la variable que contendrá la respuesta del servidor
    # $3 = tercer parámetro pasado a la función

    echo "Iniciando $nombre en puerto $puerto..."
    # echo = muestra un mensaje informativo por pantalla
    # $nombre = expande la variable nombre
    # $puerto = expande la variable puerto

    while true; do
        # while true = bucle infinito que se ejecuta continuamente
        # do = inicio del bloque de código del bucle

        echo "$respuesta" | nc -l $puerto
        # echo = muestra la respuesta del servidor
        # "$respuesta" = expande la variable respuesta (entre comillas para preservar espacios)
        # | = tubería que envía la salida de echo a nc
        # nc -l = netcat en modo servidor (listen)
        # $puerto = expande la variable puerto

    done &
    # done = fin del bloque de código del bucle
    # & = ejecuta el bucle completo en segundo plano

}
